6        Interface (Tabs and Commands)

EPS has invested a significant amount of effort to make EPPORAs Graphical User Interface (GUI) as intuitive as possible.  A guiding light in our GUIs design is to minimize the number of clicks a user has to make to obtain his/her desired results.  Each page, after the Login page, contains a menu bar with four tabs containing commands for reports, projects, administration and help.  Each page also contains a project drop-down list along with a set of tasks for the user relative to projects plus any messages generated for the user from the EPPORA system.  Section 6 describes the Login and Home pages along with the tabs and commands common to all EPPORA pages except the dialogues.

It is necessary to login to start the EPPORA system.  Figure 3 depicts an EPPORA Login page and Table 29 provides a description of each of the Login pages objects.  The Login page is invoked when a user selects the EPPORA icon from the desktop.

6.2    Home Page

6.3    Tabs

This section describes the tabs that launch the EPPORA processes.  The tab commands can be invoked from all EPPORA pages except the Login page and information, diagnostic and dialogue pages.

6.3.2             Projects Tab

The Projects Tab provides an interface for (1) Administrators to create new projects and delete existing projects, (2) Business Analysts to create/maintain their requirements and Project Managers to create/maintain schedules for their respective projects, (3) Project Managers map their low-level requirements (those with no sub-requirements) to the appropriate project tasks and (4) Project Portfolio Managers to oversee, update and approve the work of their Business Analysts and Project Managers.

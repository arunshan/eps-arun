5     Best Practices

The “Level” by default is set to the same as the preceding task’s level.  You can change it to be anywhere between 0 and the preceding task’s level plus one (signifying it is a child of the previous task).  If you do not change the level, then it is a sibling of the previous task.

Schedule Page

21

Estimated Effort

If this is a low-level task, “Estimated Effort” is the total hours for all the specified resources.  If it is not a low-level task, then EPPORA automatically determines the number of hours for this parent task by accumulating its children tasks “Estimated Efforts.”

8

Schedule Page

21

Start Date

If this is the initial task, then it will display today’s date.  If you click on the “Start Date” for this field, a calendar will appear and you can select the date from the calendar for this task’s start date.  If you leave it blank and there are dependencies, then EPPORA will automatically determine the start date.  If there are no dependencies it will be set to the start date if you do not specify a date.  You can only specify dates for low-level tasks (those without children tasks). 

9

Schedule Page

21

Finish Date

The “Finish Date” is normally calculated automatically by EPPORA by using the “Start Date” and the “Effort Level.”  If you enter the “Finish Date” first, then EPPORA will automatically calculate the “Start Date” using the “Finish Date” and the “Effort Level.”  The “Finish Date” must be greater than or equal to the “Start Date.”

Schedule Page

   D = Deliverable

   F = Fixed date (set by EPPORA)

   FD = Foreign Dependency (set by EPPORA)

   MGR = Missing Generic Resource (set by EPPORA)

   M = Milestone

   L = Late Task (set by EPPORA)

   LL = Child (low-level) Task (set by EPPORA)

   P = Parent Task (set by EPPORA)
